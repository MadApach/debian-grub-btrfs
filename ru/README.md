# Набор скриптов для настройки неубиваемой связки Btrfs + Snapper + GRUB

Данный репозиторий содержит инструменты для автоматизации продвинутой настройки файловой системы Btrfs в Debian (Testing/Sid). Скрипты решают проблемы стандартных пакетов (баг `grub-btrfs.path`, невозможность загрузки в read-only снимки) и обеспечивают стабильную работу системы восстановления.

## Структура папки и описание скриптов

Каждый шаг пронумерован и должен выполняться последовательно. 

### 1. `1_tune_btrfs_subvols.sh` — Оптимизация структуры subvolume
* **Что делает:** Переносит каталоги `/var/cache` и `/var/tmp` в отдельные сабволюмы Btrfs.
* **Зачем нужно:** Исключает временные файлы, кэши apt и flatpak из системных снимков. Это предотвращает раздувание размера снапшотов и сохраняет их «чистыми».

### 2. `2_setup_btrfs_tools.sh` — Установка и базовая настройка Snapper
* **Что делает:** Устанавливает пакеты `snapper`, `btrfs-assistant` и `curl`. Автоматически инициализирует конфигурацию Snapper для корня (`/`). 
* **Особенности:** Полностью отключает создание снимков по расписанию (`TIMELINE`), активирует создание снимков при каждой загрузке системы (`boot-timer`) и запускает интерактивный конфигуратор APT-снимков.

### ── `tune_apt_snapshots.sh` — Переключатель APT-снапшотов (Вспомогательный)
* **Что делает:** Вызывается автоматически из скрипта №2. Предоставляет интерактивное меню в терминале.
* **Зачем нужно:** Позволяет одной командой включить или выключить создание автоматических парных снимков (Pre/Post) перед выполнением `apt upgrade` или `apt install`. Полезно, если нужно временно отключить снапшоты при установке большого количества мелких пакетов.

### 3. `3_setup_grub-btrfs.sh` — Чистая установка GRUB-BTRFS
* **Что делает:** Клонирует актуальную версию `grub-btrfs` напрямую из официального репозитория GitHub, компилирует, интегрирует в систему и корректно удаляет за собой все временные файлы сборки.

### 4. `4_setup_grub_watcher.sh` — Исправление автообновления меню GRUB
* **Что делает:** Полностью отключает штатный демон `grub-btrfsd.service`. Вместо него создает кастомную связку из `systemd.path` и `systemd.service` для принудительного обновления меню GRUB при любых операциях со снапшотами.
* **Зачем нужно:** Стандартная служба в Debian не работает, так как `Inotify` ядра Linux не умеет отслеживать API-вызовы Snapper при создании сабволюмов. Этот скрипт решает проблему «слепого» загрузчика.

### 5. `5_setup_overlayroot.sh` — Обеспечение загрузки в read-only снимки (Boot to RAM)
* **Что делает:** Устанавливает и переводит в режим ожидания утилиту `overlayroot`. Интегрирует специальный параметр `overlayroot=tmpfs` в конфигурацию генерации меню `grub-btrfs`, после чего пересобирает образы `initramfs` для всех ядер и обновляет GRUB.
* **Зачем нужно:** Снимки Snapper по умолчанию создаются в режиме «только для чтения» (read-only), из-за чего обычная загрузка в них невозможна (системе необходимо писать логи, временные файлы и т.д.). Этот скрипт заставляет ядро при выборе снимка в GRUB разворачивать виртуальную файловую систему в ОЗУ поверх снимка. Система успешно загружается, работает, но любые изменения исчезают после перезагрузки.

## Как использовать

1. Сделайте скрипты исполняемыми:
   ```bash
   chmod +x *.sh
   ```
2. Запускайте строго по порядку от 1 до 5 с правами суперпользователя:
   ```bash
   sudo ./1_tune_btrfs_subvols.sh
   sudo ./2_setup_btrfs_tools.sh
   sudo ./3_setup_grub-btrfs.sh
   sudo ./4_setup_grub_watcher.sh
   sudo ./5_setup_overlayroot.sh
   ```
